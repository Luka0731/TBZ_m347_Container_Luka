#### docker compose up Hintergrundbefehle
| Befehl | Erklärung |
|---|---|
| `docker network create` | Erstellt das definierte Netzwerk |
| `docker volume create` | Erstellt allfällige Volumes |
| `docker build` | Baut Images die ein Dockerfile verwenden |
| `docker pull` | Pullt Images die direkt angegeben sind (z.B. mariadb:latest) |
| `docker create` | Erstellt die Container |
| `docker start` | Startet die Container |

#### Fehler auf db.php
Der Fehler entsteht weil das Web-Image aus KN02 den Datenbank-Hostnamen hardcoded hat (z.B. `kn02b-db`). Der neue Container heisst aber `m347-kn04b-db`, den alten Namen kennt er nicht mehr.
Lösung: Den Hostnamen in der `db.php` auf den neuen Containernamen `m347-kn04b-db` anpassen und das Image neu builden und pushen.
